## ⚡ Présentation du projet

**MiroFish** est un moteur de prédiction IA de nouvelle génération basé sur la technologie multi-agents. En extrayant des informations de base du monde réel (actualités de dernière minute, projets de lois, signaux financiers), il construit automatiquement un monde numérique parallèle à haute fidélité. Dans cet espace, des milliers d'agents intelligents dotés de personnalités indépendantes, de mémoire à long terme et de logique comportementale interagissent librement et évoluent socialement. Vous pouvez injecter dynamiquement des variables depuis une « vue divine » pour déduire précisément les trajectoires futures — **répétez l'avenir dans un bac à sable numérique et gagnez vos décisions après d'innombrables simulations**.

> Il vous suffit de : télécharger des matériaux de base (rapports d'analyse de données ou histoires de romans intéressantes) et décrire vos besoins de prédiction en langage naturel</br>
> MiroFish vous renverra : un rapport de prédiction détaillé et un monde numérique haute fidélité profondément interactif

### Notre vision

MiroFish se consacre à la création d'un miroir d'intelligence collective qui cartographie la réalité. En capturant l'émergence collective déclenchée par les interactions individuelles, nous dépassons les limites de la prédiction traditionnelle :

- **Au niveau macro** : Nous sommes un laboratoire de répétition pour les décideurs, permettant de tester politiques et relations publiques sans aucun risque
- **Au niveau micro** : Nous sommes un bac à sable créatif pour les utilisateurs individuels — que ce soit pour déduire la fin d'un roman ou explorer des scénarios imaginatifs, tout est amusant, ludique et accessible

De la prédiction sérieuse à la simulation ludique, nous permettons à chaque « et si » de voir son résultat, rendant possible la prédiction de toute chose.

## 🔄 Flux de travail

1. **Construction du graphe** : Extraction des données de base & Injection de mémoire individuelle et collective & Construction GraphRAG
2. **Mise en place de l'environnement** : Extraction des relations entre entités & Génération de personas & Injection des paramètres de simulation par l'Agent de configuration
3. **Lancement de la simulation** : Simulation parallèle sur deux plateformes & Analyse automatique des besoins de prédiction & Mise à jour dynamique de la mémoire temporelle
4. **Génération du rapport** : Le ReportAgent dispose d'un ensemble d'outils riche pour interagir en profondeur avec l'environnement post-simulation
5. **Interaction approfondie** : Dialoguez avec n'importe quel agent du monde simulé & Interagissez avec le ReportAgent

## 🚀 Démarrage rapide

### Option 1 : Déploiement depuis le code source (Recommandé)

#### Prérequis

| Outil | Version | Description | Vérification |
|-------|---------|-------------|-------------|
| **Node.js** | 18+ | Environnement d'exécution frontend, inclut npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Environnement d'exécution backend | `python --version` |
| **uv** | Dernière version | Gestionnaire de paquets Python | `uv --version` |

#### 1. Configurer les variables d'environnement

```bash
# Copier le fichier de configuration exemple
cp .env.example .env

# Éditer le fichier .env et renseigner les clés API nécessaires
```

**Variables d'environnement requises :**

```env
# Configuration de l'API LLM (supporte toute API LLM au format OpenAI SDK)
# Recommandé : modèle Qwen-plus d'Alibaba via la plateforme Bailian : https://bailian.console.aliyun.com/
# Consommation élevée, essayez d'abord des simulations de moins de 40 tours
LLM_API_KEY=votre_clé_api
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Configuration Zep Cloud
# Le quota mensuel gratuit suffit pour une utilisation simple : https://app.getzep.com/
ZEP_API_KEY=votre_clé_api_zep
```

#### 2. Installer les dépendances

```bash
# Installation en une commande de toutes les dépendances (racine + frontend + backend)
npm run setup:all
```

Ou installation étape par étape :

```bash
# Installer les dépendances Node (racine + frontend)
npm run setup

# Installer les dépendances Python (backend, crée automatiquement l'environnement virtuel)
npm run setup:backend
```

#### 3. Démarrer les services

```bash
# Démarrer le frontend et le backend simultanément (exécuter depuis la racine du projet)
npm run dev
```

**Adresses des services :**
- Frontend : `http://localhost:3000`
- API Backend : `http://localhost:5001`

**Démarrage individuel :**

```bash
npm run backend   # Démarrer uniquement le backend
npm run frontend  # Démarrer uniquement le frontend
```

### Option 2 : Déploiement Docker

```bash
# 1. Configurer les variables d'environnement (identique au déploiement source)
cp .env.example .env

# 2. Télécharger l'image et démarrer
docker compose up -d
```

Par défaut, le fichier `.env` du répertoire racine est lu et les ports `3000 (frontend) / 5001 (backend)` sont mappés.

> Une adresse miroir pour un téléchargement plus rapide est fournie en commentaire dans `docker-compose.yml`, à remplacer si nécessaire.
